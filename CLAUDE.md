# Project Forge AI Guidelines

Siempre priorizar:

1. Código limpio.

2. SOLID.

3. Evitar duplicación.

4. ECS cuando sea posible.

5. Mantener el código modular.

6. No introducir dependencias innecesarias.

7. Nunca modificar documentación automáticamente.

8. Preguntar antes de cambiar la arquitectura.

9. Todo sistema debe ser desacoplado.

10. El servidor es autoritativo.

11. Nunca confiar en el cliente.

12. Todo código nuevo debe ser documentado.

13. Todo sistema nuevo debe tener pruebas.

14. Las armas son procedurales.

15. No existen clases.

16. El equipamiento define el gameplay.

17. La creatividad del jugador tiene prioridad sobre recetas fijas.
